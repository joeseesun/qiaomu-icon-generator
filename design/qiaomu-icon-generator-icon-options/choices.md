# qiaomu-icon-generator icon test notes

Generated with Codex built-in image generation on 2026-06-21. Original generated PNG files remain in Codex's default `generated_images` directory for this run; repository copies are normalized to 1024px square PNGs.

## Prompt strategy

- Generate 7 original square app-icon candidates for a reusable icon generation skill.
- Avoid text, letters, numbers, pseudo-text, watermarks, real brand marks, and copied app icon compositions.
- Emphasize generate -> compare -> choose -> export.
- Use a Qiaomu-compatible palette: charcoal black, warm amber, porcelain white, restrained cyan.

## Shortlist

| Option | Direction | Verdict |
|---|---|---|
| 01 | Candidate grid plus cursor | Strong workflow metaphor, but visually busy at 32px. |
| 02 | Selected tile plus export arrow | Good export signal, but icon mask and arrow compete. |
| 03 | Gallery plus selected check tile | Polished, but check mark can read like generic task app. |
| 04 | Selected diamond inside reference frame | Attractive, but less specific to icon generation. |
| 05 | Source candidates flowing into final icon | Selected: best balance of meaning, bold silhouette, and 32px readability. |
| 06 | Foundry mark plus magnifying ring | Nice design-tool tone, but magnifier becomes tiny. |
| 07 | Export sizes from one selected tile | Most explanatory, but too line-heavy at favicon size. |
